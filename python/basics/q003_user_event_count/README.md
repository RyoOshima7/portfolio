# q003_user_event_count - ユーザー行動ログのイベント件数集計

## 目的
行動ログを読み込み、イベントごとの件数を集計する基礎を身につけることを目的としています。

## 使用技術
Python, csv, 条件分岐, 辞書, 関数

## 処理の流れ

CSVファイルを1行ずつ読み込み、各行の`event`列から`view`、`click`、`purchase`を取得します。取得したイベントが集計対象に含まれている場合は、辞書に保存した対応する件数を1ずつ増やし、最後にイベントごとの集計結果を表示します。

## 実行方法
1. `learning/main.py` を自力で完成させます。
2. `completed/main.py` と見比べて差分を確認します。
3. 必要なら `python learning/main.py` または `python completed/main.py` を実行します。
4. ログデータから件数を数える考え方を README と docs で確認します。

## 学び
- ログ集計はプロダクト分析の基礎であり、まずは件数を正しく数えることが重要です。


## BtoB SaaSでどう活かせるか

イベント件数を集計する考え方は、BtoB SaaSのユーザー行動分析でよく使われます。たとえば、ログイン数、クリック数、機能利用回数、資料ダウンロード数、問い合わせ数などを数えることで、ユーザーがどの機能を使っているかを把握できます。

この課題で扱った `view / click / purchase` の集計は、実務では「画面閲覧 → 機能利用 → 問い合わせ・契約」のようなファネル分析や、アクティブ率・機能利用率の確認に応用できます。

## 面接で説明するなら

単に件数を数えるだけでなく、「どの行動が多いか」「どこで次の行動に進めていないか」を見ることで、プロダクト改善の仮説につなげられると説明できます。
