# BtoB SaaS KPI Dashboard Design

## 目的

BtoB SaaSのデータ分析職を想定し、PM、CS、BizOpsが意思決定しやすいKPIダッシュボードの設計を整理することを目的としています。

現時点では実装前の設計メモですが、今後はLooker Studio、Power BI、Tableauなどで可視化することを想定しています。

## 想定ユーザー

- PM: プロダクト改善の優先順位を判断する
- CS: 解約リスクが高い顧客や利用低下している顧客を見つける
- BizOps / SalesOps: 商談化率、契約化率、売上推移を確認する
- 経営・マネージャー: MRR、解約率、継続率などの主要KPIを見る

## ダッシュボードで答えたい問い

- 今月のMRRは伸びているか
- 新規契約、アップセル、解約のどれがMRRに影響しているか
- どの機能がよく使われているか
- どの顧客セグメントで利用率が低いか
- どの段階でユーザーが離脱しているか
- CSが優先対応すべき顧客は誰か

## 主要KPI

| 分類 | KPI | 見る目的 |
|---|---|---|
| 売上 | MRR / ARR | 月次・年次の売上推移を確認する |
| 継続 | 解約率 / 継続率 | 顧客が継続しているかを確認する |
| 利用 | アクティブ率 / 機能利用率 | プロダクトが使われているかを確認する |
| ファネル | view → click → conversion | どこで離脱しているかを確認する |
| CS | 利用低下顧客数 / リスク顧客数 | 優先対応すべき顧客を見つける |

## 画面構成案

### 1. サマリーエリア

- 今月のMRR
- 前月比
- 新規契約数
- 解約数
- アクティブ率
- 解約率

### 2. ファネルエリア

- view数
- click数
- conversion数
- click率
- conversion率

### 3. 顧客セグメント別エリア

- プラン別MRR
- 企業規模別の利用率
- 業種別の解約率
- 利用頻度別の顧客数

### 4. CSアクションエリア

- 直近30日で利用が減った顧客
- 解約リスクが高い顧客
- オンボーディング未完了の顧客
- 優先フォロー候補

## ダッシュボードを見る人が取るアクション

- PM: 離脱が大きい画面や機能を改善候補にする
- CS: 利用低下している顧客にフォロー連絡を行う
- BizOps: 商談化率や契約化率の低い段階を確認する
- 経営・マネージャー: MRRの増減要因を把握する

## BtoB SaaSデータ分析職での説明ポイント

この設計では、単にKPIを並べるのではなく、**誰が、何を判断するために見るか** を意識しています。

面接では、以下のように説明できます。

- MRRや解約率だけでなく、利用ログやファネル指標も一緒に見ることで、売上変化の原因仮説を立てられる
- CS向けには、全体平均よりも「どの顧客に対応すべきか」が分かる画面が重要
- PM向けには、機能利用率や離脱ポイントを見ることで、改善施策の優先順位を考えられる
