## 用語集

| 用語                | かんたん説明                                                                     |
|:--------------------|:---------------------------------------------------------------------------------|
| 足切り              | 最低点。ここを超えないと次に進めないライン。                                     |
| Proof               | できた証拠。リンク、画像、PDFなど「見せられる形」。                              |
| Git/GitHub          | 作ったコードや資料を保存して、変更の記録も残す場所。                             |
| コミット            | GitHubに「ここまでやった」を記録して保存すること。                               |
| README              | 成果物の説明書。何をしたか・どう見るかを書く。                                   |
| Notion              | メモや資料をまとめるアプリ（ノート）。                                           |
| SQL                 | データベースから必要な情報を取り出すための言葉。                                 |
| JOIN                | 別々の表（テーブル）をつなげて1つにして見る方法。                                |
| 集計                | 合計や平均など、数字をまとめて比べること。                                       |
| CTE                 | SQLで途中の結果に名前をつけて、読みやすくする書き方。                            |
| Window関数          | 行ごとに順位や平均との差などを計算するSQLの機能。                                |
| pandas              | Pythonで表データを扱うための道具。                                               |
| EDA                 | まずデータの全体を見て、特徴をつかむこと。                                       |
| 欠損                | データが空（抜けている）ところ。                                                 |
| 外れ値              | 周りと比べて極端に大きい/小さい値。                                              |
| 仮説                | 「こうかもしれない」という予想。                                                 |
| 検証                | 仮説が合っているか、データで確かめること。                                       |
| 相関                | 一緒に増えたり減ったりする関係（原因とは限らない）。                             |
| 因果                | 原因と結果の関係。                                                               |
| KPI                 | 成果を見るための数字。例：継続率、購入数など。                                   |
| North Star          | 一番大事にする指標（北極星指標）。                                               |
| KPIツリー           | 大事な指標を、原因になりそうな小さい指標に分けた図。                             |
| ダッシュボード      | 数字を見やすくまとめて表示する画面。                                             |
| Tracking Plan       | 何を計測するかの設計書（イベント名、項目、ルールなど）。                         |
| イベント/プロパティ | イベント＝行動。プロパティ＝その行動の情報。                                     |
| NULL                | データが空の状態。                                                               |
| コホート分析        | 同じ時期に始めた人たちをまとめて、変化を見る分析。                               |
| ファネル分析        | 途中で減る場所を探す分析（例：見る→申し込む→購入）。                             |
| チャーン            | 解約。サービスをやめること。                                                     |
| リテンション        | 継続。使い続けている割合。                                                       |
| A/Bテスト           | AとBの2案を比べて、どちらが良いか確かめる方法。                                  |
| p値                 | 差がたまたま起きた可能性の目安。                                                 |
| 効果量              | どれくらい良くなったかの大きさ。                                                 |
| 信頼区間            | 本当の値がこの範囲にありそう、という幅。                                         |
| SRM                 | 実験でA/Bの人数配分が偏る問題。                                                  |
| 多重検定            | たくさん比べると偶然当たる確率が上がる問題。                                     |
| DiD                 | 前後×比較群で差を見る方法（差の差）。                                            |
| 準実験              | A/Bができないときに、できる範囲で因果に近づける方法。                            |
| raw/mart            | raw＝生データ。mart＝分析しやすく整えたデータ。                                  |
| dbt                 | データを整える処理を管理しやすくする道具。                                       |
| データ辞書          | データの意味・作り方・単位などを書いた説明書。                                   |
| 1pager              | 1枚で結論を伝える資料。                                                          |
| STAR                | 話し方の型：状況→課題→行動→結果。                                                |

| 匿名化              | 名前などを消して、特定されないようにすること。                                   |

| GRR                 | 売上の継続率（追加購入は入れない）。解約の痛さを見る。                           |

| アカウント          | B2Bでの“会社”の単位。1社に複数ユーザーがいる。                                   |
| シート（席）        | 1社の中の利用者枠。増減が売上や利用に直結することが多い。                        |
| PQL                 | Product Qualified Lead。プロダクトの利用状況から“営業すると良さそう”な見込み客。 |
| オンボーディング    | 最初に使い方を覚えて“使える状態”になるまでの流れ。                               |
| ヘルススコア        | 解約しそうかを“使い方のサイン”で点数化した目安（簡易でOK）。                     |
