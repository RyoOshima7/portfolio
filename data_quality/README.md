# Data Quality（データ品質）

## 目的
NULL、重複、参照整合性などの観点からデータ品質を確認し、壊れたデータを見つけて直す力を示すことを目的としています。

## 使用技術
SQL, データ品質チェック, テスト設計, Markdown

## 実行方法
1. `tests/` のチェック例を確認します。
2. 必要に応じて `incident_notes.md` に不具合内容、原因、再発防止策を記録します。
3. 新しいケースを追加する際は、何を守るためのチェックかを書きます。

## 学び
- データ品質は分析の前提であり、問題を早く見つけて再発を防ぐ視点が重要です。
