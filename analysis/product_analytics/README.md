# Product Analytics（プロダクト分析）

## 目的
BtoB SaaSのプロダクト分析を想定し、**問い → 仮説 → 指標 → 検証 → 示唆** の型でミニプロジェクトを蓄積することを目的としています。

## 使用技術
SQL, Python, KPI設計, ファネル分析, リテンション分析, チャーン分析, Markdown

## 実行方法
1. 各テーマごとにフォルダを作成します。
2. `README.md` に目的、データ前提、使う指標、結果、示唆、次改善を記録します。
3. 必要に応じて `queries/` にSQL、`notes.md` に学びや迷いを書き残します。

## 学び
- 指標を見るだけでなく、プロダクト上の問いと結び付けて解釈することが大切です。
