# User Event Funnel Mini（ユーザー行動ログ分析ミニ成果物）

## 目的

BtoB SaaSのユーザー行動分析を想定し、簡単なイベントログから **イベント件数 → 転換率 → 改善仮説** までを整理することを目的としています。

このミニ成果物では、Python基礎課題で扱った `view / click / purchase` のログ集計を、SaaSのファネル分析に置き換えて考えます。

## 想定する業務シーン

架空のBtoB SaaSで、ユーザーが以下の流れで行動すると仮定します。

1. `view`: 画面を見る、または機能ページを開く
2. `click`: ボタンを押す、または機能を試す
3. `purchase`: 問い合わせ、トライアル申込、契約などの成果行動を行う

実務では、この流れを使って「どこでユーザーが離脱しているか」「どの行動を改善すべきか」を考えます。

## 分析の問い

- ユーザーはどの行動まで進んでいるか
- `view` から `click`、`purchase` へどれくらい進んでいるか
- 改善するなら、どの段階を優先して見るべきか

## 集計結果

### q003: イベント件数

| event | 件数 |
|---|---:|
| view | 4 |
| click | 3 |
| purchase | 2 |

### q004: 転換率

| 指標 | 値 |
|---|---:|
| view数 | 5 |
| click数 | 3 |
| purchase数 | 2 |
| click率 | 0.60 |
| purchase率 | 0.40 |

## 読み取れること

- `view` したユーザーのうち、一定数は `click` まで進んでいます。
- 一方で、`click` から `purchase` まで進むユーザーはさらに少なくなっています。
- 実務では、ここから「ボタンの文言が分かりにくいのか」「導線が長いのか」「ユーザーにとって価値が伝わっていないのか」といった仮説を立てます。

## BtoB SaaSでの活用イメージ

この分析は、以下のような場面で活用できます。

- 無料登録後の初回利用率を確認する
- オンボーディング完了率を確認する
- 機能ページ閲覧から機能利用までの転換率を見る
- トライアルから商談化・契約化までのファネルを見る
- CSが優先対応すべき離脱ポイントを見つける

## 改善提案の例

- `view` から `click` の転換率が低い場合
  - ボタン文言、導線、画面説明、CTAの位置を見直す
- `click` から `purchase` の転換率が低い場合
  - 入力項目の多さ、申込フォーム、料金説明、導入事例の見せ方を見直す
- 全体の件数が少ない場合
  - セグメント別、期間別、流入経路別に分けて追加分析する

## 追加で確認したいデータ

- ユーザー属性（企業規模、業種、プラン）
- 日別・週別のイベント数
- 新規ユーザーと既存ユーザーの違い
- どの画面・機能で発生したイベントか
- 施策実施前後の転換率
