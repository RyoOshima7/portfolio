# Analysis

## 目的
BtoB SaaS向けの分析Proofを、**問い → 仮説 → 指標 → 検証 → 示唆 → 次改善** の流れで蓄積し、分析の考え方と説明力が伝わる状態にすることを目的としています。

## 使用技術
Markdown, SQL, Python, 統計基礎, 仮説思考, KPI設計

## 実行方法
1. `product_analytics/` `ab_testing/` `causal_inference/` の順に確認します。
2. 各テーマで `README.md` やノートを読み、目的・指標・結果・示唆を追います。
3. 新しい成果物を追加する場合は、各サブフォルダで **目的 / 使用技術 / 実行方法 / 学び** をそろえて記録します。

## 学び
- 分析成果物は、SQLや数値だけでなく「何を問い、どう解釈したか」まで残すと伝わりやすくなります。
