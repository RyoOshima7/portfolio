# A/B Testing（統計×実験）

## 目的
A/Bテストの設計・検定・解釈を、第三者に1枚で説明できる形で整理し、実験設計の基礎力を示すことを目的としています。

## 使用技術
A/Bテスト設計, 仮説設定, 指標設計, 統計的検定, Markdown

## 実行方法
1. `design.md` に仮説、主要指標、サンプルサイズの考え方を整理します。
2. `analysis.md` に検定手順、結果、解釈、注意点をまとめます。
3. `summary_onepager.md` に面接や説明用の1枚要約を作成します。

## 学び
- A/Bテストは「差が出たか」だけでなく、事前の設計と指標定義が重要です。
