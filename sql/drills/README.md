## フォルダ構成
各問題は以下の構成で統一しています。

- **01_problem.md**：問題定義・条件・出力要件  
- **02_answer.sql**：SQL 実装（要約コメント付き）  
- **03_notes.md**：学び・判断基準・詰まりポイントの記録  

---

## 学習方針

- 問題文をそのまま SQL に書くのではなく、  
  **日本語の条件を分解してから WHERE 句を組み立てる**
- 可読性を重視し、  
  別名（AS）、括弧、インデントを意識して記述する
- 「なぜそう書いたか」「どこで迷ったか」を notes.md に必ず残す

---

## これまでに扱った主な内容

- SELECT / WHERE の基本構文
- AS による別名指定
- IN / OR を使った複数条件指定
- LIKE / NOT LIKE による文字列検索
- AND / OR の組み合わせと可読性の考え方

---

## このログで意識していること

- 正解を書くことよりも、**再現できる考え方を残す**
- 数か月後に見返しても、**判断理由が分かる状態**にする
- 実務でのデータ抽出・分析業務を想定した書き方をする

---

## 今後の予定

- 集計（COUNT / SUM / GROUP BY）
- サブクエリ
- JOIN（内部結合・外部結合）
- 分析でよく使うクエリパターンの整理

SQL の基礎力を土台として、  
データアナリスト／プロダクトアナリストとして必要な分析力につなげていく予定です。
